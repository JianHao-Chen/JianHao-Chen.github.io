发表于 2019-06-05 | 更新于 2019-07-04 | 分类于 JVM

看JVM代码过程中，看到Klass_vtbl和klassVtable，不明白，于是研究下它们的区别。

Klass_vtbl

Klass_vtbl定义在文件klass.hpp，代码如下：

  • Klass_vtbl没有成员变量，还定义了一个虚方法。
  • Klass继承Klass_vtbl。
  • 按照C++的规则，上面2点使得每一个 Klass 对象一定有一个 _vptr（指向虚函数表的指针），并且这个 _vptr 在内存的位置在Klass定义的变量之前。
  • 至于为什么要强制生成这个 _vptr，应该是想把这个 _vptr “固定下来”，不想 XKlass 有而 YKlass 没有 _vptr，那么在后面计算vtable位置时就不一致了。（待核实）
